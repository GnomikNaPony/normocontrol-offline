# Проработка ограничений

Дата: 2026-06-16.

## Подтверждение изменений человеком

Перед выпуском исправленных копий открывается отдельное окно проверки изменений.
В нем показываются:

- количество документов;
- количество замен;
- путь к исходному документу;
- список замен `старое значение -> новое значение`;
- количество вхождений по каждой замене.

Кнопка `Отклонить` ничего не меняет. Кнопка `Подтвердить выпуск копий` создает
только исправленные копии. Исходные документы не перезаписываются. После
применения создается отчет `report_*.md`.

## Стили, поля, таблицы, подписи, нумерация и колонтитулы

Реализовано как структурная проверка DOCX при импорте и анализе.

Программа теперь фиксирует:

- длинные абзацы без явного стиля Word;
- похожую на ручную нумерацию вместо автоматической нумерации Word;
- таблицы без строк;
- разделы без явных полей страницы;
- рисунки/объекты без подписи в том же абзаце;
- отсутствие полей Word;
- отсутствие колонтитулов Word.

Эти проверки не меняют документ автоматически. Они создают замечания категории
`style`, `numbering`, `table`, `layout`, `caption`, `fields`, `headers`, которые
должен проверить нормоконтролер.

Ограничение, которое остается: программа пока не знает точные допустимые
значения полей, стилей и подписей из каждого стандарта. Для полного контроля
нужно формализовать правила из РУ/СТО и добавить профиль требований.

## Ссылки, разбитые Word на несколько фрагментов

Ранее замена надежно работала только если вся ссылка лежала в одном XML-фрагменте
`w:t`. Теперь механизм работает в два этапа:

1. Точная XML-замена внутри одного `w:t`, которая сохраняет форматирование.
2. Дополнительная замена по объединенному тексту абзаца, если ссылка разбита на
   несколько `w:t`.

Второй режим может упростить форматирование только внутри затронутого абзаца,
поэтому применяется только после подтверждения человеком и фиксируется в отчете.

## OCR-текст

Для сканов и изображений программа добавляет предупреждение `ocr_review`.
Смысл: распознанный текст нельзя считать таким же надежным, как текстовый слой
DOCX/PDF. Если по OCR-документу нужно строить правило, человек должен сверить
фрагмент с оригинальным сканом.

Ограничение, которое остается: Tesseract может ошибаться в номерах, датах и
обозначениях. Для критичных стандартов лучше хранить ручную вычитку или
текстовую версию.

## Маленькая LLM

Локальная Qwen3-0.6B проверялась многократными прогонами. Она запускается, но
дает общие ответы и часто не формирует полезный финальный результат. Поэтому:

- LLM не имеет права автоматически менять документы;
- LLM не считается источником нормативного правила;
- формальные проверки выполняются детерминированным ядром;
- результат LLM можно использовать только как подсказку человеку.

Для улучшения качества нужен отдельный тест более сильной локальной модели и
RAG-пайплайн с выдержками из стандартов.
