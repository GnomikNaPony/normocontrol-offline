# Выбор локальных моделей

Проверено 16 июня 2026 года.

## Рекомендуемый стек

| Компонент | Выбор | Причина |
|---|---|---|
| Локальная LLM, минимальная | [Qwen3-0.6B-GGUF](https://huggingface.co/Qwen/Qwen3-0.6B-GGUF) | Apache-2.0, официальный GGUF, мультиязычная, работает через CPU |
| Локальная LLM, более свежий кандидат | [Qwen3.5-0.8B-GGUF](https://huggingface.co/lmstudio-community/Qwen3.5-0.8B-GGUF) | Apache-2.0, маленький GGUF-квант Q4, подходит для дальнейшего теста на слабом ноутбуке |
| Движок LLM | [llama.cpp](https://github.com/ggml-org/llama.cpp) | MIT, CPU/GPU, Windows/macOS/Linux, квантование 2-8 бит |
| Семантический поиск, опционально | [multilingual-e5-small](https://huggingface.co/intfloat/multilingual-e5-small) | MIT, поддерживает русский, заметно легче LLM |
| Орфография/грамматика, опционально | [LanguageTool](https://github.com/languagetool-org/languagetool) | Работает локально, поддерживает русский, но добавляет Java и увеличивает поставку |

## Решение

В базовую поставку не следует встраивать тяжелую модель. Программа должна
полностью выполнять точные проверки и обновление ссылок без LLM.

Для слабого ноутбука предлагается модель до 1B параметров в GGUF Q4/Q8. Если
памяти 8 ГБ и больше, можно тестировать `Qwen3.5-0.8B-GGUF` или модель класса
1.5-2B. Качество русской технической корректуры обязательно измеряется на
локальном приемочном наборе, а не по общим тестам модели.

Локальный прогон `Qwen3-0.6B-GGUF:Q8_0` через `llama.cpp` подтвердил запуск и
низкие требования: файл модели около 604 МБ, контекст 4096 занимал примерно
1.3 ГБ unified memory на macOS. Качество ответа на выдержках РУ.00.01/РУ.00.22
оказалось слишком общим: модель не должна сама принимать решения об исправлении
документа. Ее безопасная роль в этом проекте - объяснять найденный фрагмент и
предлагать текст замечания.

Попытка быстро проверить `Qwen3.5-0.8B-GGUF:Q4_K_M` 16 июня 2026 года была
остановлена из-за сетевого тайм-аута при скачивании модели. Кандидат оставлен
для следующего прогона, но не считается подтвержденным в текущей базе.

Тонкая настройка весов на слабом ноутбуке не нужна. Для этой задачи полезнее:

- индексировать новые стандарты и сразу использовать их при проверке;
- извлекать кандидаты правил из исправленных документов;
- подтверждать правила человеком;
- дообучать модель позже только на очищенном и достаточно большом наборе.
