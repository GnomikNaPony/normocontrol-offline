# Нормоконтроль Offline

Первый рабочий прототип локальной программы нормоконтроля. Приложение не требует
интернета во время работы, открывается в собственном окне, хранит данные в SQLite
и не изменяет исходные документы.

Подробная инструкция по установке, первому запуску и рабочим сценариям:
[ИНСТРУКЦИЯ.md](../ИНСТРУКЦИЯ.md).

## Что уже работает

- импорт DOCX, ODT, PDF, TXT/MD, изображений и старых DOC;
- OCR для сканов PDF/JPG/PNG/TIFF через локальный Tesseract;
- извлечение текста, ссылок на нормативные документы и разметки Word;
- полнотекстовый поиск по документам и стандартам;
- обучение на парах документов «до/после»;
- просмотр и ручное включение изученных правил;
- проверки пробелов, пунктуации, изученных исправлений и устаревших ссылок;
- пакетная замена ссылок с выпуском новых DOCX-копий;
- минималистичное desktop-окно и командная строка.

Исправления, извлеченные из пар «до/после», по умолчанию отключены: содержательные
изменения нельзя автоматически считать универсальными правилами. Их нужно
проверять перед активацией.

## Запуск

Требуется Python 3.11 или новее. Для OCR нужен установленный Tesseract с языками
`rus` и `eng`.

```bash
cd normocontrol_offline
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -r requirements.txt
python run.py
```

На Windows активация окружения:

```powershell
.venv\Scripts\activate
python run.py
```

Старый формат DOC импортируется через LibreOffice. На macOS также пробуется
системный `textutil`. Если конвертера нет, сохраните DOC как DOCX.

Главное окно содержит четыре действия:

- `База`: открыть существующую SQLite-базу или создать новую.
- `Добавить материалы`: загрузить документы, стандарты, примеры или схемы.
- `Проверить базу`: выполнить анализ и обновить список замечаний.
- `Обновить ссылку`: указать старое и новое обозначение стандарта.
- `Исправленные копии`: показать план замен, запросить подтверждение и
  выпустить DOCX-копии с отчетом `report_*.md`.

## Быстрая проверка через CLI

```bash
python -m normocontrol.cli --db data/base.sqlite3 init
python -m normocontrol.cli --db data/base.sqlite3 import "../документы на которых основываемся" --role standard
python -m normocontrol.cli --db data/base.sqlite3 import "../До и после изменений" --role example
python -m normocontrol.cli --db data/base.sqlite3 import "../Хороший нормоконтроль" --role document
python -m normocontrol.cli --db data/base.sqlite3 learn
python -m normocontrol.cli --db data/base.sqlite3 analyze
python -m normocontrol.cli --db data/base.sqlite3 findings
```

Пример массового обновления ссылки:

```bash
python -m normocontrol.cli --db data/base.sqlite3 map "ГОСТ Р 2.105-2019" "ГОСТ Р 2.105-2025"
python -m normocontrol.cli --db data/base.sqlite3 preview
python -m normocontrol.cli --db data/base.sqlite3 apply corrected --confirm
```

## Локальная нейросеть

Базовый продукт работает без LLM. Нейросеть подключается опционально через
локальный `llama.cpp`/`llama-server`, без внешних API и без доступа в интернет.

Проверенный минимальный кандидат: `Qwen3-0.6B-GGUF`. Она запускается на слабом
железе, но для нормоконтроля дает слишком общие ответы без жестких правил и
найденных выдержек. Поэтому массовые замены, ссылки, структура и форматирование
должны проверяться детерминированным ядром, а LLM используется только для
подсказок человеку.

Подробнее: [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md) и
[docs/MODELS.md](docs/MODELS.md). Первичная выжимка требований из загруженных
РУ: [docs/RU_REQUIREMENTS.md](docs/RU_REQUIREMENTS.md).
Кроссплатформенная проверка: [docs/CROSS_PLATFORM.md](docs/CROSS_PLATFORM.md).

## Ограничения прототипа

- Пакетное исправление сейчас выполняет только подтвержденные замены ссылок.
- Ссылки, разорванные Word на несколько форматированных фрагментов, показываются
  в отчете, но могут не замениться автоматически.
- Полный контроль полей, таблиц, нумерации и стилей будет добавлен после
  формализации требований из загруженных стандартов.
- Для старых DOC надежнее установить LibreOffice или заранее конвертировать их
  в DOCX.
