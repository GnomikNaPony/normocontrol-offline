# Подробная инструкция

Эта инструкция описывает запуск и базовую работу с прототипом
`Нормоконтроль Offline`. Программа рассчитана на локальную работу без доступа в
интернет: документы остаются на компьютере, база хранится в SQLite, исходные
файлы не перезаписываются.

## Что находится в репозитории

- `normocontrol_offline/` - исходный код программы.
- `документы на которых основываемся/` - стандарты и РУ, по которым строятся
  проверки.
- `До и после изменений/` - пары документов для извлечения кандидатов правил.
- `Хороший нормоконтроль/` - примеры документов, прошедших нормоконтроль.
- `Архивы для обучения/Плохие ИЭ/` - документы с ошибками для проверки.
- `Схемы проекта/` - фотографии схем и ручная расшифровка идеи проекта.

В репозиторий намеренно не добавлены `.venv`, локальная SQLite-база, кэши,
исправленные копии и веса нейросетей. Эти файлы создаются на конкретном
компьютере и могут быть большими.

## Что нужно установить

Минимально:

- Python 3.11 или новее.
- Tesseract OCR с языками `rus` и `eng`, если нужно читать сканы PDF и фото.
- LibreOffice, если нужно надежно импортировать старые файлы `.doc`.

Опционально:

- `llama.cpp`, если нужно подключать локальную маленькую LLM.
- GGUF-модель, например `Qwen3-0.6B-GGUF` или другой кандидат до 1B параметров.

## Установка на macOS

Откройте Terminal.

```bash
brew install python tesseract tesseract-lang
brew install --cask libreoffice
```

Проверка OCR:

```bash
tesseract --list-langs
```

В списке должны быть `rus` и `eng`.

Перейдите в папку проекта:

```bash
cd "/Users/aleksandrkiko/Desktop/Нормоконтроль Offline/normocontrol_offline"
```

Создайте окружение и установите зависимости:

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

Запустите программу:

```bash
python run.py
```

## Установка на Windows

Установите Python 3.11 или новее и включите пункт `Add Python to PATH`.

Установите Tesseract OCR с русским языковым пакетом. После установки проверьте,
что команда доступна:

```powershell
tesseract --list-langs
```

Установите LibreOffice, если планируется импорт старых `.doc`.

Откройте PowerShell в папке `normocontrol_offline` и выполните:

```powershell
py -3 -m venv .venv
.venv\Scripts\activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
python run.py
```

Если PowerShell запрещает запуск скриптов активации, временно разрешите их для
текущего пользователя:

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

## Первый запуск через окно программы

После команды `python run.py` откроется отдельное окно `Нормоконтроль`.

1. Нажмите `База`.
2. Выберите `Нет`, если нужно создать новую базу.
3. Укажите путь, например `data/normocontrol.sqlite3`.
4. Нажмите `Добавить материалы`.
5. Выберите файлы или папки с документами.
6. Укажите тип материалов.
7. После загрузки примеров нажмите `Обучить`.
8. После обучения нажмите `Проверить базу`.

Типы материалов:

- `Стандарты` - ГОСТ, РУ, СТО, постановления и другие нормативные основания.
- `Документы` - обычные рабочие документы, которые нужно проверять.
- `Примеры` - пары “до/после” и документы с уже внесенными исправлениями.
- `Схемы` - фотографии, схемы и текстовые описания проектной логики.

## Рекомендуемый порядок загрузки

Сначала загрузите стандарты:

- `документы на которых основываемся/`
- `документы на которых основываемся/РУ/`

Затем загрузите хорошие документы:

- `Хороший нормоконтроль/`

Затем загрузите пары для обучения:

- `До и после изменений/`

Затем загрузите проблемные документы:

- `Архивы для обучения/Плохие ИЭ/`

После этого нажмите `Проверить базу`. Программа обновит список замечаний,
найденных ссылок и кандидатов правил.

Если были загружены пары “до/после”, сначала нажмите `Обучить`. После обучения
программа сохранит извлеченные правила и удалит временный текст примеров из
SQLite-базы, чтобы база не росла бесконечно. Исходные документы на диске не
удаляются.

## Обновление ссылки на новый стандарт

Сценарий: вышел новый стандарт, а в базе есть документы со ссылками на старую
редакцию.

1. Нажмите `Обновить ссылку`.
2. Введите старое обозначение, например `ГОСТ Р 2.105-2019`.
3. Введите новое обозначение, например `ГОСТ Р 2.105-2025`.
4. Программа пересчитает замечания и покажет документы, где найдена старая
   ссылка.
5. Нажмите `Исправленные копии`.
6. Выберите папку для результата.
7. Проверьте список документов и количество замен в окне подтверждения.
8. Нажмите `Да`, только если изменения можно применять.

Важно: исходные DOCX не перезаписываются. Исправления выпускаются в отдельную
папку. В этой же папке создается отчет `report_YYYY-MM-DD_HH-MM-SS.md` со
ссылками на исходные документы, исправленные копии и таблицей замен.

## Работа через командную строку

Командная строка полезна для повторяемых прогонов и больших баз.

Инициализация базы:

```bash
python -m normocontrol.cli --db data/base.sqlite3 init
```

Импорт стандартов:

```bash
python -m normocontrol.cli --db data/base.sqlite3 import "../документы на которых основываемся" --role standard
```

Импорт примеров “до/после”:

```bash
python -m normocontrol.cli --db data/base.sqlite3 import "../До и после изменений" --role example
```

Импорт рабочих документов:

```bash
python -m normocontrol.cli --db data/base.sqlite3 import "../Хороший нормоконтроль" --role document
python -m normocontrol.cli --db data/base.sqlite3 import "../Архивы для обучения/Плохие ИЭ" --role document
```

Импорт схем:

```bash
python -m normocontrol.cli --db data/base.sqlite3 import "../Схемы проекта" --role scheme
```

Извлечение кандидатов правил:

```bash
python -m normocontrol.cli --db data/base.sqlite3 learn
```

По умолчанию после обучения база сжимается: из примеров удаляются абзацы,
полнотекстовый индекс, ссылки и разметка, а извлеченные правила остаются. Это
экономит место на диске. Для отладки можно временно оставить примеры:

```bash
python -m normocontrol.cli --db data/base.sqlite3 learn --keep-examples
```

Очистить обучающие примеры и сжать базу вручную:

```bash
python -m normocontrol.cli --db data/base.sqlite3 cleanup
```

Анализ базы:

```bash
python -m normocontrol.cli --db data/base.sqlite3 analyze
python -m normocontrol.cli --db data/base.sqlite3 findings
```

Статистика:

```bash
python -m normocontrol.cli --db data/base.sqlite3 stats
```

Поиск по базе:

```bash
python -m normocontrol.cli --db data/base.sqlite3 search "нормоконтроль"
python -m normocontrol.cli --db data/base.sqlite3 search "РУ.00.22"
```

Добавление замены ссылки:

```bash
python -m normocontrol.cli --db data/base.sqlite3 map "ГОСТ Р 2.105-2019" "ГОСТ Р 2.105-2025"
```

Предварительный просмотр изменений без применения:

```bash
python -m normocontrol.cli --db data/base.sqlite3 preview
```

Выпуск исправленных копий:

```bash
python -m normocontrol.cli --db data/base.sqlite3 apply corrected --confirm
```

Если выполнить `apply` без `--confirm`, программа покажет план изменений, но не
создаст исправленные копии. Это защита от случайного массового применения.

## Как программа обучается

На текущем этапе программа не дообучает веса нейросети. Это сделано специально:
на маленьком ноутбуке fine-tuning будет тяжелым, а риск испортить технический
смысл документа высокий.

Вместо этого используются более надежные механизмы:

- поиск ссылок на нормативные документы;
- полнотекстовый индекс стандартов и документов;
- извлечение кандидатов правил из пар “до/после”;
- хранение правил в SQLite;
- автоматическая очистка временного текста примеров после обучения;
- ручное подтверждение содержательных правил перед применением;
- автоматическое применение только точных подтвержденных замен.

## Локальная нейросеть

Базовая программа работает без нейросети. Локальная LLM нужна только для
пояснений и подсказок человеку.

Рекомендуемый минимальный путь:

1. Установить `llama.cpp` на компьютере с интернетом.
2. Скачать GGUF-модель до 1B параметров.
3. Перенести бинарник `llama-server` и модель на офлайн-компьютер.
4. Запустить сервер только на `localhost`.

Пример:

```bash
llama-server -m models/qwen3-0.6b.gguf --host 127.0.0.1 --port 8080 -c 4096
```

Ограничение: LLM не должна сама менять документы. Она может только предложить
замечание, объяснение или вариант исправления для проверки человеком.

## Что делать, если PDF или фото плохо распознаны

- Проверьте, что установлен `tesseract`.
- Проверьте наличие языка `rus` командой `tesseract --list-langs`.
- Для плохих сканов используйте более четкий PDF или изображение.
- Для рукописных схем лучше добавить ручную расшифровку в `.md`, как сделано в
  `Схемы проекта/описание_схем.md`.

## Что делать со старыми DOC

Старый формат `.doc` нестабилен для автоматического разбора. Если импорт выдает
ошибку, откройте файл в LibreOffice или Microsoft Word и сохраните как `.docx`.

После конвертации импортируйте новый `.docx` повторно.

## Безопасный рабочий процесс

- Храните исходные документы отдельно.
- Для каждой крупной проверки создавайте новую SQLite-базу или делайте копию
  существующей.
- Не включайте содержательные правила автоматически.
- Проверяйте исправленные DOCX-копии перед передачей в работу.
- Для массового обновления ссылок сначала выполните анализ, затем применяйте
  исправления в отдельную папку.
- Перед применением проверяйте окно подтверждения и итоговый Markdown-отчет.

## Текущие ограничения

- Полная проверка стилей, полей, таблиц, подписей, автоматической нумерации и
  колонтитулов пока не реализована.
- DOCX-замены надежнее всего работают, когда ссылка не разбита Word на
  несколько фрагментов с разным форматированием.
- OCR-текст может содержать ошибки, поэтому требования из сканов нужно
  проверять человеком.
- Маленькая LLM дает общие ответы и не заменяет формальные правила.
